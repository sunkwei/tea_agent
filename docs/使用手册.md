# Tea Agent 使用手册

> **版本** 0.10.11 · **协议** MIT · **项目地址** [github.com/sunkwei/tea_agent](https://github.com/sunkwei/tea_agent)

Tea Agent 是一款**会自我进化的 AI 编程助手**，拥有 70+ 可调用工具，能自主编写代码、调试、搜索、操控浏览器，并能在运行中动态创建新工具。支持 **GUI / CLI / Web / TUI / REST API / ACP 协议** 六种界面形态。

---

## 一、快速安装

```bash
# 从 PyPI 安装
pip install tea_agent

# 从源码安装
git clone https://github.com/sunkwei/tea_agent
cd tea_agent
pip install -e .

# Web 界面依赖（可选）
pip install starlette uvicorn python-multipart

# Playwright 浏览器（可选）
playwright install chromium
```

### 验证安装

```bash
python -c "from tea_agent import Agent; print('OK')"
```

---

## 二、配置 API Key

创建配置文件 `~/.tea_agent/config.yaml`：

```yaml
main_model:
  api_key: "sk-xxx"
  api_url: "https://api.openai.com/v1"
  model_name: "gpt-4o"
  max_context_tokens: 64000

cheap_model:
  api_key: ""
  api_url: ""
  model_name: ""
  max_context_tokens: 0

embedding:
  provider: openai
  model: text-embedding-3-small
```

在 `~/.tea_agent/` 下可存放多个 `.yaml` 配置文件，Web 界面底部下拉框一键切换。


## 四、界面详解

### 4.1 Web V2 — 最全功能

单页应用(SPA)，纯 HTML/CSS/JS，**无前端框架依赖**。浏览器访问 http://127.0.0.1:8080

| 功能 | 说明 |
|------|------|
| 💬 流式对话 | SSE 实时推送，逐 token 输出 |
| 📋 会话管理 | 左侧面板，点击切换 |
| 🧠 记忆管理 | 查看/搜索/添加/删除长期记忆 |
| ⏱️ 任务调度 | 定时任务 CRUD（cron/interval/daily） |
| 🔍 全局搜索 | 搜索聊天记录、记忆、任务 |
| 📤 PDF 导出 | 导出当前会话 |
| 🌙 主题切换 | 深色/浅色 + 强调色 |
| ⚡ 配置切换 | 底部下拉框一键切换配置 |

**并发架构：** 每请求独立 Session，无需全局锁，支持不同配置并行运行。

### 4.2 GUI 桌面 — 原生体验

基于 Tkinter 的原生客户端。

```bash
tea_agent
```

功能：实时流式对话、Markdown 渲染、会话列表、记忆面板、定时任务管理、PDF 导出、系统托盘常驻、主题切换、字体缩放。


## 六、长期记忆系统

Tea Agent 的记忆系统模拟人类记忆的**优先级分层**、**相关性检索**、**自然衰减**、**去重合并**。基于 SQLite 持久化 + embedding 语义向量。

### 6.1 记忆结构

每条记忆包含：

| 字段 | 说明 |
|------|------|
| `content` | 记忆内容（精简摘要） |
| `priority` | 优先级：0=CRITICAL / 1=HIGH / 2=MEDIUM / 3=LOW |
| `importance` | 重要度 1-5 |
| `category` | 分类：instruction / preference / fact / reminder / general |
| `tags` | 逗号分隔标签 |
| `embedding` | numpy float32 向量（语义搜索） |
| `expires_at` | 过期时间（null=永不过期） |
| `pinned` | 是否钉住（豁免年龄衰减） |

### 6.2 选择算法

每次对话开始，选出最相关的 **≤30 条** 注入上下文：

```
score = 匹配度 × 重要度 × 时效因子 × 优先级因子

时效因子: 1天内=1.0, 7天=0.9, 30天=0.7, 90天=0.5, >90天=0.3
优先级因子: (4 - priority) / 4
```

**分层保底策略：**
- CRITICAL 优先入选（上限 10 条，FIFO 取最新）
- HIGH ≥ 3 条、MEDIUM ≥ 2 条、LOW ≥ 1 条保底
- 剩余名额按 score 自由竞争

### 6.3 年龄衰减

模拟 Ebbinghaus 遗忘曲线，pinned=true 豁免：

| 原始 | 衰减条件 | 降级为 |
|------|---------|--------|
| CRITICAL | 创建 > 30 天 | HIGH |
| HIGH | 创建 > 60 天 | MEDIUM |
| MEDIUM | 创建 > 90 天 | LOW |

### 6.4 LLM 优先级精调

`llm_adjust_priorities()` 使用便宜模型评估近期主题，微调优先级：
- 只能 ±1 级调整，每次最多 3 条
- 升级时重置 created_at（重新计时）

### 6.5 自动提取

对话结束后，LLM 从用户消息中自动提取记忆：

| 分类 | 条件 | 优先级 |
|------|------|--------|
| instruction | 用户要求"记住"的规则 | CRITICAL |
| preference | 用户的习惯/偏好 | HIGH |
| reminder | 有时效的提醒 | HIGH |
| fact | 技术事实/架构决策 | MEDIUM |
| general | 其他参考 | LOW |

### 6.6 去重合并

写入前执行去重流水线：Jaccard 相似度 ≥ 0.3 → 合并更新（保留更长 content、取更高优先级、tags 并集）。

### 6.7 格式化注入

```python
if priority == CRITICAL:   return "!!! 必须遵循:"
if category == "reminder": return "⏰ 提醒:"
if category == "preference": return "💡 偏好:"
if category == "fact":     return "📌 事实:"
return "📎"
```

Agent 可通过 `toolkit_memory` 工具手动管理记忆。

## 七、历史压缩机制

Tea Agent 使用**四级分层**构建发送给 LLM 的上下文，在有限 token 窗口内最大化信息密度。

```
┌─ Level 0: 系统层 ──────────────────────────────┐
│  系统提示词 + 技能注入 + 记忆注入 + TODO 恢复      │
├─ Level 3: 摘要层（LLM 生成）─────────────────────┤
│  L2 溢出时生成：保留关键结论，丢弃细节              │
├─ Level 2: 历史对列表（SQLite 持久化 50 条）──────┤
│  user+AI final msg 对，相关性筛选动态注入           │
├─ Level 1: 最新对话（当前 session）───────────────┤
│  压缩工具链 + 旧工具输出→占位符 + 渐进式裁剪        │
└─────────────────────────────────────────────────┘
```

### L1 压缩

三层防线：
1. **实时截断：** 工具输出 >128KB 时保留首尾各 64KB
2. **旧工具占位符：** 3 轮外的 tool 消息 → `[工具结果已省略: N 字符]`
3. **渐进式裁剪：** 超出 token 预算时 5 级裁剪

### L2 历史对

SQLite 环形缓冲区，容量 50 条。写入时若溢出，最老 30 条送给 L3 摘要，保留最新 20 条。

**相关性筛选：** 用 Jaccard 相似度匹配当前对话关键词：
- ≥ 0.15 → 保留完整 user+assistant 对
- ≥ 0.05 → 仅保留摘要片段
- < 0.05 → 不注入（保底：最高分的一条）

### L3 语义摘要

L2 溢出时，用便宜模型将 30 轮对话（≈20K tokens）压缩为 ~500 tokens 摘要，包含：项目背景、已完成修改、关键决策、架构约束、用户偏好、待办事项。

## 八、自进化引擎

Tea Agent 的**自进化体系**由五个层次构成。

### 8.1 工具热插拔

```python
# 运行时创建新工具
toolkit_save(name="my_tool", meta={...}, pycode="...")
toolkit_reload()  # 立即生效

# 安全回滚
toolkit_list_versions("my_tool")
toolkit_rollback("my_tool", "1.0.0")
```

每次 save 自动递增版本号，保存历史版本，生成 SKILL.md 文档。

### 8.2 五层安全自修改

Agent 修改自身代码时的安全防护链：

```
Layer 0: Git 快照（git commit "snapshot: pre-evolve"）
Layer 1: 时间戳 .bak 文件（永不覆盖历史备份）
Layer 1.5: Python 语法严格检查（缩进/括号/冒号）
Layer 2: py_compile 编译验证（失败→自动回滚）
Layer 2.5: LSP 智能检查（影响分析+lint+签名对比）
Layer 3: pytest 测试验证（失败→git reset --hard）
```

### 8.3 提示词进化

Agent 可以**自我优化系统提示词**：
- `list`: 查看所有版本历史
- `evolve`: 基于反思+记忆→LLM 生成新版本→设为活跃
- `rollback`: 回滚到指定版本

### 8.4 经验固化

任务完成后自动复盘：
- 成功 → `solidify` → 固化到技能库
- 失败 → `lesson` → 记录到经验库

```python
toolkit_experience_solidify(action='auto', task='...', result='...', success=True)
```

### 8.5 后台进化线程

每小时自动运行：
1. 工具使用率分析→优化建议
2. docs/TOOLS.md 同步
3. 技能模式整理

```python
toolkit_self_evolve_thread(action='start')
toolkit_self_evolve_thread(action='status')
```

## 九、多 Agent 系统

### 9.1 分治并发

`toolkit_parallel_subtasks` 将复杂问题分解为子任务并行执行：

```python
toolkit_parallel_subtasks(subtasks=[
    {"id": "task_1", "description": "分析文件A", "difficulty": "easy"},
    {"id": "task_2", "description": "分析文件B", "difficulty": "easy"},
    {"id": "task_3", "description": "整合结果", "difficulty": "hard"},
])
```

- easy → Lite Agent 并发执行
- hard → 主 Agent 执行

### 9.2 子 Agent 系统

`toolkit_subagent` 生成隔离上下文的子 Agent：

```python
toolkit_subagent(action='spawn_sync', goal='分析 git log', max_iterations=10)
toolkit_subagent(action='spawn', goal='...')  # 异步
toolkit_subagent(action='list')               # 查看所有子 Agent
```

### 9.3 Dispatcher 多步骤

```python
from tea_agent.multi_agent import Dispatcher

dispatcher = Dispatcher()
result = dispatcher.dispatch("重构项目添加类型注解")
print(result["summary"])
```

自动识别任务模式、生成子任务列表、拓扑排序分层执行。

### 9.4 辩论赛 Demo

```bash
python demo/multi_agent/server.py --port 8083
# 浏览器打开 http://127.0.0.1:8083
```

双 AI 对抗辩论，50 轮实时交替。甲乙双方独立选择配置文件与模型，左右分屏 SSE 流式推送。

## 十二、上下文窗口控制

`max_context_tokens` 用于限制发送给 LLM 的最大上下文 token 数：

| 值 | 适用场景 |
|:--:|----------|
| 0 | 不限制，发送全部历史 |
| 32000 | 适合 32K 窗口模型（如 DeepSeek） |
| 64000 | 默认，适合 64K~128K 窗口 |
| 128000 | 适合 GPT-4o / Claude 大窗口模型 |

启用后自动渐进式裁剪：
1. 删除旧的 `[历史记录]` 条目
2. 替换旧工具输出为占位符
3. 清空 thinking 内容
4. 截断长文本
5. 删除旧轮次（保留最近 5 轮）

主模型和便宜模型独立配置，互不影响。

## 十三、Ruff 代码规范（v0.10.11+）

`pyproject.toml` 内置 Ruff 配置，确保代码风格统一：

```toml
[tool.ruff]
line-length = 150
target-version = "py310"

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "UP", "B", "C4", "SIM"]
```

| 规则集 | 说明 |
|--------|------|
| `E` / `W` | pycodestyle 错误/警告 |
| `F` | pyflakes 逻辑错误 |
| `I` | isort 导入排序 |
| `N` | pep8-naming 命名规范 |
| `UP` | pyupgrade Python 3.10+ 语法升级 |
| `B` | flake8-bugbear 常见 bug 检测 |
| `C4` | 化简代码 |
| `SIM` | 简化表达式 |

所有源码已通过 Ruff 检查，采用 Python 3.10 现代类型注解（`str | None` 替代 `Optional[str]`）。

## 十四、MCP 协议支持

Tea Agent 支持作为 MCP (Model Context Protocol) 客户端，连接外部 MCP Server：

```python
# 连接文件系统 MCP Server
toolkit_mcp(
    action='connect',
    server_name='filesystem',
    command='npx',
    args=['-y', '@modelcontextprotocol/server-filesystem', '/path/to/allow']
)

# 列出可用工具
toolkit_mcp(action='list_tools', server_name='filesystem')

# 调用工具
toolkit_mcp(action='call_tool', server_name='filesystem',
    tool_name='read_file',
    tool_args={'path': '/tmp/test.txt'})

# 查看状态 / 断开
toolkit_mcp(action='status')
toolkit_mcp(action='disconnect', server_name='filesystem')
```

支持 stdio 和 SSE 两种传输方式。

## 十五、常见问题

### Q: 启动时报 "No module named 'tea_agent'"？

A: 确保已安装：`pip install -e .`（源码）或 `pip install tea_agent`（PyPI）。

### Q: 如何切换模型？

A: 修改 `~/.tea_agent/config.yaml` 中的 `model_name` 和 `api_key`。Web 界面底部可直接切换配置。

### Q: 运行中卡住或超时？

A: 检查 `max_iterations`（默认 25）和 `max_context_tokens`（默认 64000）。可通过 `toolkit_config` 调整：
```
toolkit_config(action='set', key='max_iterations', value='50')
```

### Q: 如何查看 Agent 当前状态？

A: 使用 `toolkit_self_report()` 查看工具数量等状态信息。

### Q: 记忆可以手动管理吗？

A: 可以。GUI/Web 界面都有记忆管理面板，Agent 也可通过 `toolkit_memory` 工具管理。

### Q: 如何清理所有聊天记录？

A: 删除数据库文件重新启动：`rm ~/.tea_agent/chat_history.db`（会自动重建）。

### Q: GUI 界面无法启动？

A: 确保系统有 Tkinter：Windows 自带，Linux 需 `sudo apt install python3-tk`。

### Q: 如何贡献工具？

A: 在对话中让 Agent 用 `toolkit_save` 创建 → 自动保存到 `tea_agent/toolkit/` 目录。或直接向项目提交 PR。

### Q: 工具调用报 "参数解析失败"？

A: v0.10.11 已大幅修复 JSON 解析容错。如仍出现，检查参数格式：键名和字符串值必须用双引号，特殊字符需转义。可通过 `toolkit_config(action='set', key='tool_exec_max_retries', value='3')` 增加重试次数。

### Q: 如何将 Mini 版编译为单文件？

A: 使用 `python build_nuitka.py` 将 Mini 版编译为单文件可执行文件（`.exe` / ELF）。需要安装 Nuitka 和 C 编译器，编译耗时约 5-30 分钟。详见第十章。
