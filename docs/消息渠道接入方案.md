### 1.2 关键设计原则

| 原则 | 说明 |
|------|------|
| **平台无关** | 消息网关层屏蔽各平台差异，Core Server 无需感知消息来源 |
| **会话隔离** | 每个平台用户映射到独立 tea_agent topic_id，互不干扰 |
| **异步非阻塞** | 使用 asyncio + 消息队列，不阻塞平台消息接收 |
| **失败隔离** | 一个平台故障不影响其他平台 |
| **插拔式** | 新增平台只需实现 PlatformAdapter 接口，注册到网关 |

#### 微信签名验证

微信签名验证通过 URL 参数中的 signature、timestamp、nonce 进行：

```python
async def verify_signature(self, raw_body: bytes, signature: str) -> bool:
    """验证微信签名：SHA1(sorted(token, timestamp, nonce))"""
    from urllib.parse import parse_qs
    params = parse_qs(signature)
    arr = sorted([self.token, params.get("timestamp", [""])[0],
                  params.get("nonce", [""])[0]])
    sign = hashlib.sha1("".join(arr).encode()).hexdigest()
    return sign == params.get("signature", [""])[0]
```

#### 消息格式转换

微信使用 XML 格式，需要解析为统一消息：

```python
async def parse_incoming(self, raw: dict, headers: dict) -> Optional[IncomingMessage]:
    """解析微信 XML 消息"""
    xml_str = raw.get("xml", {}) if isinstance(raw, dict) else {}
    if not xml_str:
        return None

    msg_type = xml_str.get("MsgType", "")
    from_user = xml_str.get("FromUserName", "")
    to_user = xml_str.get("ToUserName", "")

    if msg_type == "text":
        return IncomingMessage(
            platform="wechat_mp",
            platform_user_id=from_user,
            platform_chat_id=from_user,
            text=xml_str.get("Content", ""),
            raw=raw,
        )
    elif msg_type == "image":
        # 图片消息：需通过素材 API 下载
        media_id = xml_str.get("MediaId", "")
        pic_url = xml_str.get("PicUrl", "")
        return IncomingMessage(
            platform="wechat_mp",
            platform_user_id=from_user,
            platform_chat_id=from_user,
            text=xml_str.get("Content", xml_str.get("MsgId", "")),
            raw=raw,
        )
    return None
```

#### 回复消息

```python
async def send_message(self, chat_id: str, msg: OutgoingMessage) -> bool:
    """发送客服消息（48 小时内可主动推送）"""
    token = await self._get_access_token()
    url = f"https://api.weixin.qq.com/cgi-bin/message/custom/send?access_token={token}"
    payload = {
        "touser": chat_id,
        "msgtype": "text",
        "text": {"content": msg.text},
    }
    async with httpx.AsyncClient() as client:
        resp = await client.post(url, json=payload)
    return resp.json().get("errcode") == 0
```

### 4.2 企业微信（WeCom）

#### 企业微信 Bot（单向通知）

```python
class WeComBotAdapter(PlatformAdapter):
    """企业微信 Bot（仅支持发送，不支持接收）"""

    def __init__(self, webhook_key: str):
        self.webhook_url = f"https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key={webhook_key}"

    async def send_message(self, chat_id: str, msg: OutgoingMessage) -> bool:
        """发送 Markdown 消息"""
        payload = {"msgtype": "markdown", "markdown": {"content": msg.text}}
        async with httpx.AsyncClient() as client:
            resp = await client.post(self.webhook_url, json=payload)
        return resp.json().get("errcode") == 0

    async def send_action(self, chat_id: str, action: str) -> None:
        pass  # 企业微信 Bot 不支持 action

    async def parse_incoming(self, raw: dict, headers: dict) -> Optional[IncomingMessage]:
        return None  # Bot 无法接收消息

    async def verify_signature(self, raw_body: bytes, signature: str) -> bool:
        return True
```

> **注意**：企业微信 Bot 是单向通道。如需双向交互，应使用**企业微信自建应用**（需额外开发回调接口）。

## 5. 通用 Webhook 接入点

### 5.1 路由注册

在 `tea_agent/server/server.py` 的 `create_app()` 中添加：

```python
# 注册消息渠道 Webhook 路由
from tea_agent.channel.webhook_handler import handle_channel_webhook

routes.append(Route(
    "/api/channel/webhook/{platform}",
    endpoint=handle_channel_webhook,
    methods=["POST", "GET"],  # GET 用于微信验证
))
```

### 5.2 配置格式

```yaml
# config.yaml 扩展 - 消息渠道配置
channel:
  # Telegram Bot
  telegram:
    enabled: false
    bot_token: ""                      # Bot Token (必填)
    webhook_url: ""                    # Webhook URL (必填，需 HTTPS)
    secret_token: ""                   # Secret Token (推荐)
    allowed_users: []                  # 空=允许所有，非空=白名单
    max_message_length: 4000

  # 微信公众号
  wechat_mp:
    enabled: false
    app_id: ""
    app_secret: ""
    token: ""
    encoding_aes_key: ""

  # 企业微信
  wecom:
    enabled: false
    webhook_key: ""                    # Bot 的 Webhook Key
    corpid: ""                         # 自建应用时填写
    agentid: ""
    callback_token: ""

  # 全局设置
  global:
    auto_confirm_max_iter: true        # max_iter 自动确认
    question_timeout: 300              # question 超时秒数
    default_parse_mode: "markdown"
    notify_on_error: true              # 出错时通知用户
```

## 6. 安全性设计

### 6.1 平台签名验证

| 平台 | 验证方式 | 实现 |
|------|----------|------|
| Telegram | `X-Telegram-Bot-Api-Secret-Token` Header | header 比对 |
| 微信公众号 | SHA1(token, timestamp, nonce) URL 参数签名 | 参数排序 + SHA1 |
| 企业微信 | AES 加解密 + 回调 URL Token | AES-256-CBC |

### 6.2 用户授权级别

| 级别 | 说明 | 适用场景 |
|------|------|----------|
| **开放模式** | 任何平台用户均可使用 | 个人部署、Demo |
| **白名单模式** | 仅允许指定用户 ID 列表 | 团队内部使用 |
| **密码认证** | 首次使用发送密码验证 | 半公开 Bot |
| **API Key 模式** | 配合 Tea Agent --api-key | 与 Server 认证联动 |

### 6.3 操作限制

- **命令黑名单**：禁止渠道用户触发危险工具（如 `toolkit_exec` 删除文件）
- **超时控制**：单次 Agent 调用超时自动中断（默认 300s）
- **频率限制**：单用户每分钟最多 20 条消息
- **内容过滤**：敏感词过滤（可选）

## 7. 部署方案

### 7.1 方案 A：集成部署（推荐）

消息网关作为 Server 内置模块：

```bash
# 启动 Server（自动加载 channel 配置）
python -m tea_agent.server --host 0.0.0.0 --port 8080

# 配置 Telegram Webhook（首次安装后执行）
python -m tea_agent.channel.setup_webhook --platform telegram
```

### 7.2 方案 B：独立部署

消息网关作为独立进程：

```bash
# 终端 1: Core Server（监听内网）
python -m tea_agent.server --host 127.0.0.1 --port 8080

# 终端 2: Message Gateway（对外暴露 Webhook）
python -m tea_agent.channel --config config.yaml --port 9090
```

### 7.3 Docker 部署

```yaml
# docker-compose.yml 扩展
version: "3.8"
services:
  tea-agent:
    build: .
    container_name: tea-agent
    restart: unless-stopped
    ports:
      - "8081:8081"           # Core Server
      - "9090:9090"           # Webhook 入口（可选，独立部署时）
    environment:
      - TEA_CHANNEL_TELEGRAM_ENABLED=true
      - TEA_CHANNEL_TELEGRAM_BOT_TOKEN=${BOT_TOKEN}
      - TEA_CHANNEL_TELEGRAM_WEBHOOK_URL=https://bot.example.com/api/channel/webhook/telegram
    volumes:
      - tea_agent_data:/root/.tea_agent
```

### 7.4 Nginx 配置（Telegram Webhook 必须 HTTPS）

```nginx
server {
    listen 443 ssl http2;
    server_name bot.example.com;

    ssl_certificate     /etc/letsencrypt/live/bot.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/bot.example.com/privkey.pem;

    location /api/channel/webhook/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
