# 测试验证流程

## 非 GUI 修改验证流程
> 优先级: HIGH | 重要度: 5

每次修改核心逻辑后，按以下流程验证：

### 1. 语法/导入检查
```bash
python -c "from tea_agent.tea_main_cli import TeaCLI"
python -c "import ast; ast.parse(open('tea_agent/xxx.py').read())"
```

### 2. 单元测试
```bash
python -m pytest test_session.py -v
```

### 3. 功能验证（需 API Key）
```bash
python -m tea_agent.tea_main_cli --oneshot "1+1等于几？"
```

### 4. 交互调试
```bash
python -m tea_agent.tea_main_cli --debug
```

## test_main.py 说明
- `test_main.py` 原指向 `main_db_gui.py`（GUI），超时是预期行为
- 应改为导入 `tea_main_cli` 的 `TeaCLI` 做纯逻辑验证
- GUI 相关测试应人工执行
